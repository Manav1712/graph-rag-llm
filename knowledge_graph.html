Knowledge Graph Visualization Guide

What is this?

This is an interactive knowledge graph visualization showing relationships between entities extracted from your documents. The graph represents concepts, people, places, and events connected by their relationships.

Understanding the Visualization

Nodes (Circles)

  • Size: Larger nodes have more connections (higher degree)
  • Color: Different colors represent different entity types:
    • Red - Events (actions, processes)
    • Teal - People (individuals, characters)
    • Blue - Geographic locations (cities, places)
    • Green - Organizations (companies, groups)
    • Yellow - Other/Unknown types
  • Labels: Show the entity name next to each node

Links (Lines)

  • Thickness: Thicker lines indicate stronger relationships (higher weight)
  • Color: Gray lines that become more opaque on hover
  • Meaning: Each line represents a relationship between two entities

Interactive Features

Navigation

  • Zoom: Scroll to zoom in/out
  • Pan: Click and drag to move around the graph
  • Drag Nodes: Click and drag individual nodes to reposition them

Information Display

  • Hover over nodes: See detailed information in a tooltip
  • Click on links: View relationship details in a popup
  • Settings panel: Customize the visualization appearance

Configuration Options

Layout Styles

  • Web-like: Balanced spacing, natural clustering
  • Compact: Nodes closer together, dense layout
  • Spread Out: More space between nodes, easier to read
  • Circular: Attempts to arrange nodes in a circle

Visual Customization

  • Node Size: Choose between degree-based or fixed size
  • Link Thickness: Based on relationship weight or fixed
  • Color Scheme: Color by entity type or degree
  • Labels: Show all labels, only connected nodes, or none

Data Interpretation

What the Data Represents

This knowledge graph was created from your document collection. The entities and relationships were automatically extracted using natural language processing techniques.

Example: If you see a large red node labeled "VECTOR" connected to a smaller blue node "SCALAR PRODUCT" with a thick line, this indicates that "VECTOR" has many connections and a strong relationship with "SCALAR PRODUCT" in your documents.

Understanding Metrics

  • Degree: Number of connections a node has
  • Weight: Strength of the relationship between entities

Tips for Exploration

  • Start by looking for the largest nodes - these are your most important concepts
  • Follow thick lines to find the strongest relationships
  • Use the settings to adjust the view for different analysis needs
  • Try different color schemes to highlight different aspects of your data
  • Zoom in on dense areas to see individual relationships clearly
